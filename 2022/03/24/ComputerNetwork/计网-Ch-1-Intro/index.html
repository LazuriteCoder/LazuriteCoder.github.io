1.1 基本概念

计算机网络？ 网络协议？ 网络分类？

网络

计算机网络是通信技术与计算机技术紧密结合的产物。

定义：计算机网络是互联的、自治的计算机集合。

  • 互联——互联互通
    • 通信链路
  • 自治——无主从关系

计算机网络通过交换网络连接主机。

一些QA：

Q: 什么是 Internet

A: 从组成细节角度来看：

  • 全球最大的互联网络

从服务角度看：

  • 为网络应用提供通信服务的通信基础设施
  • 为网络应用提供应用编程接口

协议

协议是什么？

协议——计算机网络中的数据交换必须先遵守事先约定好的规则。

网络协议 (network protocol)，简称为协议 ，是为进行网络中的数据交换而建立的规则、标准或约定。

协议 规定了通信实体之间所交换的消息的 格式、意义、顺序以及针对收到信息或发生的事件所采取的“动作”(actions)

协议的三要素

  • 语法 (Syntax)
    • 数据与控制信息的结构或格式
    • 信号电平
  • 语义 (Semantics)
    • 需要发出何种控制信息
    • 完成何种动作以及做出何种响应
    • 差错控制
  • 时序 (Timing)
    • 时间顺序
    • 速度匹配

协议规范了网络中所有信息的发送和接收过程。

网络分类

  • 按照网络覆盖范围分类：
    • 个人区域网PAN (Personal Area Network) ~10米
    • 局域网LAN (Local Area Network) ~1公里
    • 城域网MAN (Metropolitan Area Network) ~5-50公里
    • 广域网WAN (Wide Area Network) ~几十-几千公里
  • 按照网络使用者分类：
    • 专用网(private network)
    • 公用网(public network)
  • 按照网络拓扑分类：
    • 总线网络
    • 星形网络
    • 树形网络
    • 网状网络

1.3 数据交换

电路交换、报文交换、分组交换 数据交换是数据通过网络核心从源主机到达目的主机的方式。

三种交换形式

电路交换：

  • 最典型的电路交换网络：电话网络
  • 电路交换的三阶段
    • 建立连接
    • 通信
    • 释放连接
  • 独占资源

报文交换是一种将信息整体发送的形式。

分组是报文分拆出来的一系列相对较小的数据包，进行分组交换需要将报文进行拆分和重组，这意味着它将产生额外开销。

报文交换 vs 分组交换

首先引入传输延迟的概念：

\(delay = \dfrac{L(bits)}{R(bits/sec)}\)

  • 拆分为较小长度为 \(L bits\) 的分组(packets)
  • 在传输速率为 \(R\) 的链路上 传输分组

那么来看一个报文交换和分组交换的例子：

报文交换

  • 报文长度 \(M\) bits
  • 链路带宽为 \(R\) bps
  • 每次传输报文需要 \(M/R\) 秒

分组交换：

  • 报文被拆分为多个分组
  • 分组长度为 \(L\) bits
  • 每个分组传输时延为 \(L/R\) 秒

eg:
配一张图，大概是
PC-设备-设备-PC
\(M = 7.5 Mbits, L = 1500 bits (M = 5000L)\)
\(R = 1.5 Mbps\)
报文交换的交付时间为:
\(M / R = 7.5 / 1.5 = 5 s\)
总时间为：\(5 \times 3 = 15 s\)。
分组交换的交付时间为:
因为已经分了 5000 组，一个组传输一段的用时为 \(L/R = 1500/(1.5 \times 10^6) = 1 \ ms\)
想象一个这样的情况，从第一组开始，用时 1ms 出发到达第一个中途交换机，这样第二组也可以开始出发了，那么到第 5000 组出发的时候，已经经过了 4999 个 1ms，第 5000 组再走 3 段路就可以全部送达了！所以总共需要：
\(1ms \times (4999 + 3) = 5002ms\)
\(5002 ms = 5.002 s\)

总结: 一个分组交换的报文交付时间如下：

  • 报文：\(M \ bits\)
  • 链路带宽：\(R \ bps\)
  • 分组长度（大小）：\(L \ bits\)
  • 跳步数：\(h\)
  • 路由器数：\(n\)

那么有： \(T = M/R + (h-1)L/R = M/R + nL/R\)

例题again:
H1->设备0->设备1->H2;
H1->设备0->设备2->设备1->H2
在下图所示的采用“存储-转发”方式的分组交换网络中，所有链路的数据传输速率为 100 Mbps，分组大小为 1 000 B，其中分组头大小为 20 B。若主机H1向主机H2发 送一个大小为 980 000 B 的文件，则在不考虑分组拆装时间和传播延迟的情况下，从H1发送开始到 H2 接收完为止，需要的时间至少是？
首先计算一下分组：
巧了，正好分组大小和分组头大小组合一下，就可以让 980 B 的文件和 20 B 的分组头一起凑成 1 000 B，那么正好可以分成 1000 组。
那么整个文件的延迟：\(M/R = (1000 \times 1000B) \times 8 / (100 \times 10^6 bps) = 80ms\)
有一个路由选择基本原理，所有数据分组应该经过两个路由器的转发，用时最少，所以总时间只需要再加上最后一个分组的两次转发延迟，即为：
\(2 \times (2 \times 1000B) \times 8 / (100 \times 10^6 bps) = 0.16ms\)
于是总时间：\(80+0.16=80.16ms\)

1.4 性能指标

速率、带宽、时延、时延带宽积、吞吐量

速率和带宽

速率是数据传输速率或是比特率

带宽是信号具有的频带宽度，最高频率与最低频率之差（原本定义） 但是网络中的带宽是指数字信道传送的最高数据率

延迟/时延

Q: 分组交换为什么会发生时延和丢包？ A: 分组在路由器缓存中排队

  • 分组到达速率超出链路容量时
  • 分组排队，等待输出链路可用

四种分组延迟：

\(d_{nodal} = d_{proc} + d_{queue} + d_{trans} + d_{prop}\) \(d_{proc}\):

节点处理延迟

  • 差错检测
  • 确定输出链路
  • 通常 < msec

\(d_{queue}\): 排队延迟

  • 等待输出链路可用
  • 取决于路由器拥塞程度

\(d_{trans}\): 传输延迟

  • \(L\): 分组长度 (bits)
  • \(R\): 链路带宽 (bps)
  • \(d_{trans} = L/R\)

\(d_{prop}\): 传播延迟

  • \(d\): 物理链路长度
  • \(s\): 信号传播速度 (~2*10^8m/sec)
  • \(d_{prop}=d/s\)

利用率

\(U = \dfrac{t_{PKT}}{t_{PKT} + RTT + t_{ACK}}\)

分组丢失（丢包）

  • 队列缓存容量有限
  • 分组到达已满队列将被丢弃（即丢包）
  • 丢弃分组可能由前序结点或源重发（也可能不重发） \(丢包率=\dfrac{丢包数}{已发分组总数}\)

吞吐量/率

  • 吞吐量:表示在发送端与接收端之间传送数据速率 (b/s)
    • 即时吞吐量: 给定时刻的速率
    • 平均吞吐量: 一段时间的平均速率
